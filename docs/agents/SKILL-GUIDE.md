# Skill 新手操作手册

> 学习时间：约15分钟 | 不需要编程基础

---

## 一、什么是 Skill？

**Skill = 一份工作说明书**，告诉 AI 在什么情况下做什么。

就像：
- 老板给员工写 SOP
- 你给 AI 写一套"遇到这种情况就这么做"的规则

---

## 二、Skill 的结构

```
skill-name/
└── SKILL.md        ← 核心文件，就是这份"说明书"
```

**只有一样东西是必须的：SKILL.md**

其他都是可选的（脚本、例子等），我们先不管。

---

## 三、SKILL.md 是什么？

就是一个 **Markdown 格式的文本文件**，分成两部分：

### 1. 头部信息（固定格式）

```markdown
---
name: 技能名字
description: 简短描述。用于：用户说xxx时
---
```

### 2. 主体内容

```markdown
# 技能名字

## 这个技能做什么

## 工作流程

1. 第一步
2. 第二步
3. 第三步

## 输出格式

[你希望输出的样子]
```

## 四、5分钟快速实战

### 场景：做一个"翻译助手" Skill

**目标**：用户说"翻译一下"、"帮我翻译"，AI 就帮你翻译文本

**步骤**：

1. 创建文件夹
2. 创建 SKILL.md
3. 填入内容

---

### Step 1：创建文件夹

在终端运行：
```bash
mkdir -p ~/.claude/skills/my-translator
```

### Step 2：创建 SKILL.md

用任意编辑器创建文件：
```bash
nano ~/.claude/skills/my-translator/SKILL.md
```

### Step 3：写入内容

```markdown
---
name: translator
description: 中英文互译，翻译文档、段落、单词。用在用户说"翻译"、"帮我翻译英文"、"这句话英文怎么说"时。
---

# 翻译助手

## 功能

- 中译英
- 英译中
- 保持专业术语准确

## 工作流程

1. 用户提供要翻译的文本
2. 确定原文语言
3. 翻译成目标语言
4. 保留专业术语（如有）

## 输出格式

**原文**：...
**译文**：...

## 注意

- 专业术语需要准确
- 保持原文语气
- 中英混杂时分别处理
```

### Step 4：保存并测试

按 `Ctrl+O` 保存，`Ctrl+X` 退出。

在 Claude Code 中说：`/translator` 或 `翻译一下 "Hello world"`

## 五、常见问题

### Q：Skill 放在哪里？
A：放在 `~/.claude/skills/` 目录下

### Q：如何让 Skill 生效？
A：重启 Claude Code 或输入 `/skills reload`

### Q：一个 Skill 能做多件事吗？
A：可以，但在 description 里列清楚触发场景

### Q：需要写代码吗？
A：不需要！Skill 就是纯文本，用自然语言写即可

### Q：怎么删除 Skill？
A：删除 `~/.claude/skills/技能名/` 整个文件夹

## 六、SKILL.md 头部说明

```markdown
---
name: my-skill          # 技能名字，用英文或中文都可以
description: 简短描述。# 必须写！告诉AI什么时候用这个技能
---
```

**description 格式**：
```
做xxx。用在：用户说xxx时、遇到xxx情况时。
```

**好例子**：
```
整理会议记录，提取决策和待办。用在：用户说"整理会议"、"会议记录"时。
```

**差例子**：
```
帮助处理文档。
```

## 七、工作流程写法

### 简单版（3步以内）
```markdown
## 工作流程
1. 用户输入内容
2. 转换成目标格式
3. 输出结果
```

### 详细版（复杂任务）
```markdown
## 工作流程

1. **接收输入** — 用户提供原始内容
2. **分析处理** — 理解需求，判断格式
3. **执行转换** — 按规则处理内容
4. **输出结果** — 返回处理好的内容

## 检查清单
- [ ] 输入有效？
- [ ] 格式正确？
- [ ] 结果完整？
```

## 八、输出模板写法

## 输出格式

### 格式A（简单）
- 原始内容：[原文]
- 转换结果：[结果]

### 格式B（表格）
| 字段 | 内容 |
|------|------|
| 名称 | xxx |
| 日期 | xxx |

### 格式C（结构化）
```json
{
  "field1": "value1",
  "field2": "value2"
}
```
```

## 十、快速检查表

创建 Skill 时检查：

- [ ] `name` 字段：简洁、好记
- [ ] `description` 字段：写清楚什么时候用
- [ ] 工作流程：步骤清晰
- [ ] 输出格式：有模板/示例
- [ ] 文件位置：`~/.claude/skills/技能名/SKILL.md`
